## **저는 웹개발자입니다**

웹 프론트엔드 개발자로서 저의 역할은 혁신적인 비즈니스 아이디어를 구현함으로써 사람들에게 가치를 제공하며, 사회적으로 유용한 영향을 주는 것입니다.

프론트엔드 개발자는 주로 사용자의 유저 플로우와 상호작용을 개선하는 작업을 담당합니다. 사용자가 웹 어플리케이션을 원활하게 이용할 수 있도록, 직관적이고 효과적인 UI/UX를 구현하는 것이 제 목표입니다. 그 외에도 백엔드 서버와 데이터를 주고받아 처리하며, 캐싱, 부드러운 사용자 플로우, 애니메이션, 웹앱 성능 및 번들 크기 최적화와 같은 다양한 측면에도 신경을 씁니다. 

### **다시 돌아온 웹의 시대**
 
UI/UX 툴 피그마(Figma)를 사용해보신 적이 있으신가요? 대학생 분들이라면 미리캔버스를 많이 써보셨을 텐데요. 데스크탑 앱을 설치할 필요 없이 웹에서 바로 사용 가능하죠. 화상 회의를 해야 할 때 앱을 설치해야 하는 줌(Zoom)보다는 웹링크만 누르면 켜지는 구글 미트(Google Meet)가 더 편하지 않으신가요? 플러그인의 다양화, 웹 클라이언트 시스템이 고도화되면서 새로운 웹의 시대가 도래했습니다.

`앱`과 `웹` 모두 모바일 및 디지털 환경에서 서비스와 콘텐츠를 제공하는 주요 플랫폼입니다. 하지만 최근에는 웹이 앱보다 더욱 중요한 플랫폼으로 인식되고 있으며, 다양한 분야에서 웹 애플리케이션의 개발과 사용이 확대되고 있습니다. 크로스 플랫폼 호환성(다양한 운영체제나 디바이스에서 동작), 쉬운 업데이트와 배포, 유리한 검색 엔진 최적화, 접근성과 공유성 측면에서 웹은 앱보다 빠르게 변하는 비즈니스 생태계에 적응하기 유리해졌기 때문입니다.

## **웹 프로젝트 경험을 통해 내가 얻은 것들**

### (1) **웹 서비스의 성능**

웹 개발자로서(특히 프런트엔드 개발자로서) 웹 서비스의 성능은 매우 중요합니다. 사용자는 빠르고 반응성 있는 웹 사이트를 원하며, 웹 성능이 좋지 않으면 사용자가 이탈할 가능성이 높아지기 때문입니다. 그래서 프런트엔드 개발자는 웹 성능을 높이기 위해서 몇 가지 중요한 요소를 염두에 두고 개발을 진행합니다.

  - 로딩 속도 최적화: 웹 페이지의 로딩 속도는 사용자 경험에 직접적인 영향을 미칩니다. 프런트엔드 개발자는 최적화된 이미지, 압축된 리소스, 적절한 캐싱 등을 활용하여 로딩 속도를 개선해야 합니다.

  - 반응성과 성능 최적화: 웹 페이지는 사용자와의 상호작용에 빠르게 반응해야 합니다. 프런트엔드 개발자는 JavaScript와 CSS를 효율적으로 사용하여 사용자 인터페이스를 빠르고 부드럽게 만들어야 합니다. 또한, 불필요한 네트워크 요청을 최소화하고, 비동기적인 데이터 로딩을 활용하여 성능을 향상시켜야 합니다.

  - 모바일 최적화: 웹 사이트는 모바일에서도 원활하게 작동해야 합니다. 프런트엔드 개발자는 반응형 웹 디자인, 모바일 최적화 기술 등을 사용하여 모바일 사용자의 경험을 개선해야 합니다.

  - 코드 최적화: 프런트엔드 개발자는 효율적이고 최적화된 코드를 작성해야 합니다. 불필요한 코드나 중복된 작업을 제거하고, 메모리 누수를 방지하며, 성능에 영향을 주는 부분을 개선해야 합니다.

  - 성능 모니터링과 테스팅: 프런트엔드 개발자는 또한 성능 모니터링과 테스팅을 통해 웹 사이트의 성능을 지속적으로 확인하고 문제를 해결합니다. 성능 분석 도구를 사용하여 로딩 시간, 페이지 성능, 네트워크 요청 등을 측정하고, 발생하는 문제를 식별하고 최적화를 진행합니다.
  
### (2) **방문자 행동 추적 및 분석**

웹 서비스의 성공은 방문자 수의 증가와 선순환 구조의 구축에 달려있습니다. 사용자들에게 가치 있는 콘텐츠와 서비스를 제공하는 것은 필수적인 요소입니다. 그러나 수익 극대화를 위해서는 최적화 전략의 적용이 필요합니다. 웹 분석 도구를 통해 방문자의 행동을 추적하고 분석하여 서비스 개선에 활용할 수 있습니다. 이를 통해 수익을 다시 서비스 향상에 투자하고 사용자 경험을 개선함으로써 선순환 구조를 유지하고 성장할 수 있습니다.

이를 위해 프런트엔드 개발자는 가설을 세웁니다. 의사가 환자를 진찰할 때 면담, 관찰, 검사 등을 통해 환자의 상태와 문제를 파악하고 자신의 지식과 경험을 바탕으로 그 현상의 원인을 찾는 것처럼, 개발자도 비슷한 과정을 거칩니다. 예를 들어 A 버튼의 클릭 수가 기대보다 적다는 현상을 발견했다고 가정해 보겠습니다. 개발자는 왜 이런 현상이 발생했는지 여러 가설을 세우게 됩니다. A 버튼이 필요 없는 기능인가요? 필요한데 사용자가 눈에 띄지 않는 건가요? A 대신 B라는 버튼을 사용하면 어떨까요? A를 건너뛰고 다음 단계로 바로 넘어가는 것은 어떨까요? 그리고 다양한 방법을 동원하여 여러 가설 중 정답을 찾습니다. B 버튼의 클릭 수와 A 버튼의 클릭 수 사이의 상관관계를 분석하거나, 버튼의 디자인과 위치를 변경하여 테스트해 보거나, 사용자와 직접 인터뷰하거나 사용자의 행동을 관찰합니다. 이렇게 진정한 원인을 찾아내고 '무엇'과 '왜'를 정의하게 되면 문제를 해결할 준비가 된 것입니다.

따라서, 프런트엔드 개발자는 웹 서비스의 성공을 위해 사용자 경험과 성능을 개선하기 위해 가설을 세우고, 데이터 분석과 테스트를 통해 원인을 찾고 문제를 해결하는 데 주력해야 합니다. 이를 통해 웹 서비스는 지속적으로 발전하고 선순환 구조를 구축하여 성공을 이뤄낼 수 있습니다.
​
### (3) **개발도 중요하지만 기획도 중요하다**

웹 서비스를 개발하는 과정을 흔히 건물을 짓는 일에 비유하곤 합니다. 하나의 건물을 짓기 위해서는 설계, 건축가, 시공사와 같은 역할이 필요합니다. 마찬가지로 웹 서비스를 만들기 위해서는 기획자, 디자이너, 개발자와 같은 다양한 역할이 필요합니다. 작은 규모의 토이 프로젝트에서는 기획, 디자인, 개발 등을 혼자 혹은 소규모 팀으로 진행해 보는 것이 일반적입니다. 하나의 서비스 또는 기능을 만들기 위해 전체 작업 과정을 경험해 보는 것이죠.

그러다 보니 개발에 착수하기 전에 해당 기능이 우리 서비스에 왜 필요한지, 어떤 역할을 수행해야 하는지, 다른 요소들과의 관계는 어떠한지, 어떤 디자인을 적용해야 하는지 등을 판단해야 합니다. 사용자와 상호작용하는 모든 요소뿐만 아니라 숨겨진 부분까지도 고려해야 합니다. 웹 서비스를 사용할 때 무심코 넘어가는 작은 버튼이나 페이지의 움직임 하나하나도 기획 단계의 고민과 의도가 담긴 결과물입니다.

모든 결정을 직감과 감정에 의존하기만 할 수는 없습니다. 물론 업무 속도와 효율을 고려할 때 감각적인 판단이 필요한 경우도 있습니다. 하지만 보다 성공 확률을 높이기 위해서는 정량적인 데이터 분석을 통해 문제를 정의하는 것이 필요합니다. 이를 위해서는 다양한 종류의 데이터를 수집하고 연관시켜 유의미한 정보를 추출하는 '유의미한 사고' 능력이 필요합니다. 

만약 혼자가 아닌 여럿이서 하는 팀 프로젝트라면 커뮤니케이션 능력이 중요해집니다. 무엇을, 왜 만들어야 하는지 설명하고 설득하기 위해 '잘 말해야' 합니다. 그렇게 여러 번의 조율을 통해 합의된 내용을 '누구나' 이해하기 쉽고 명확하게 잘 써서 문서화(documentation)하는 것도 중요한 능력이죠.

### (4) 시각화에 대한 감각

프런트엔드 개발자로서 중요한 역량 중 하나는 '시각화에 대한 감각'입니다. 백엔드에서 제공하는 유의미한 데이터를 사용자가 쉽고 정확하게 활용할 수 시각화 요소를 디자인해야 합니다.  시각적으로 매력적이여야 사용성과 사용자 만족도를 높이는 데 도움을 줍니다.

시각화는 사용자뿐만 아니라 팀 내 구성원의 커뮤니케이션과 협업을 강화하는 데에도 도움을 줍니다. 개발자는 디자이너, 기획자, 백엔드 개발자 등 다양한 팀원들과 함께 작업하게 되는데, 우수한 시각화를 통해 개발자는 아이디어를 효과적으로 전달하고, 팀원들과의 의사소통을 원활하게 할 수 있습니다.

### (5) 효과적인 방법, 효율적인 일정관리

우선, 개발에 있어서 효과적인 방법을 찾는 것은 프로젝트의 생산성과 품질에 직접적인 영향을 미칩니다. 개발 과정에서 효율적인 코드 작성, 적절한 설계와 아키텍처 선택, 최신 개발 도구와 기술의 활용 등입니다. 

또한, 효율적인 일정관리는 프로젝트를 원활하게 진행하고 마감일을 지킬 수 있도록 도와줍니다. 일정을 체계적으로 계획하고 추적함으로써 작업 우선순위를 결정하고 리소스를 효율적으로 할당할 수 있습니다. 

클라이언트나 팀원들로부터 신뢰를 받기 위해서는 일정을 지키고 품질 좋은 결과물을 제공하는 것이 필수적입니다. 효과적인 방법을 찾고 효율적으로 일정을 관리하는 역량은 팀과 조직 내에서의 신뢰와 성과를 얻는 데 필수적입니다.
